# คู่มือการทดสอบระบบแบบ End-to-End (E2E Testing Guide)

คู่มือฉบับนี้จัดทำขึ้นเพื่อให้ทีมงานหรือผู้ทดสอบสามารถลองใช้งานระบบ **Batch Organization Creation** ได้ด้วยตนเองตั้งแต่ต้นจนจบ (ตั้งแต่อัปโหลดไฟล์ไปจนถึงข้อมูลเข้าฐานข้อมูลจริง)

---

## 🛠 0. สิ่งที่ต้องเตรียมก่อนเริ่มทดสอบ (Prerequisites)
หากคุณทดสอบบนเครื่อง Local ของตัวเอง (Local Development) ตรวจสอบให้แน่ใจว่า:
1. **รันฝั่ง Backend:** เปิด Terminal ไปที่โปรเจกต์ `fondue-org-importer` แล้วรันคำสั่ง `go run cmd/api/main.go`
2. **รันฝั่ง Frontend:** เปิด Terminal ไปที่โปรเจกต์ `org-chart-creation` แล้วรันคำสั่ง `npm run dev`
3. เข้าใช้งานผ่านเบราว์เซอร์ที่ http://localhost:5173

*(หากทดสอบผ่านลิงก์ Staging/Production ที่ Deploy แล้ว สามารถข้ามขั้นตอนนี้และเปิดหน้าเว็บได้เลย)*

---

## 🚀 เฟส 1: การนำเข้าข้อมูลเบื้องต้น (Data Ingestion)
1. เปิดหน้าเว็บขึ้นมา ระบบจะถามหา **รหัสผ่าน (API Key)**
   - ให้กรอกรหัสผ่าน (ค่าเริ่มต้นคือ: `traffy-batch-org-import`)
2. ในหน้าแรกของการสร้างผัง จะมี 2 ทางเลือก:
   - **อัปโหลดไฟล์ CSV/Excel:** กดเลือกไฟล์ตัวอย่างจากเครื่องของคุณ
   - **ใช้ Google Sheets:** คัดลอกลิงก์ Google Sheets มาวางแล้วกด "ดึงข้อมูล"
3. เมื่อข้อมูลถูกโหลดเข้ามาในตาราง ให้กดปุ่ม **"ตรวจสอบโครงสร้าง" (Check Structure)**
4. ระบบจะทำการตรวจสอบความถูกต้อง (เช่น มีลูปไหม, มีใครไม่มีหัวหน้าไหม) หากผ่าน ให้กดปุ่ม **"ส่งออก Backend" (Send to Backend)**

---

## 🔍 เฟส 2: การตรวจสอบความซ้ำซ้อน (Similarity Check)
เมื่อกดส่งออก Backend ระบบจะยิง API ไปเช็คชื่อหน่วยงานทั้งหมดกับฐานข้อมูลจริง
1. หากพบชื่อที่ "คล้ายกัน" (Similarity >= 40%) จะมี **Modal แจ้งเตือนความซ้ำซ้อน** เด้งขึ้นมา
2. ใน Modal นี้ ข้อมูลจะถูกแบ่งออกเป็น 4 ระดับความคล้าย (90-100%, 75-89%, 60-74%, 40-59%)
3. ให้คุณลองกดพิจารณาทีละแถว:
   - **ผูกข้อมูลเดิม (LINK):** เลือกว่าจะใช้ ID เดิมที่มีอยู่ในระบบแล้ว
   - **สร้างใหม่ (CREATE):** ยืนยันที่จะสร้างหน่วยงานนี้เป็น ID ใหม่
4. เมื่อรีวิวครบถ้วนและไม่มีข้อขัดแย้ง (Conflict) หลงเหลือแล้ว ให้กด **"ยืนยันและดำเนินการต่อ"**

---

## 📤 เฟส 3: ส่งข้อมูลเข้าตระกร้าพัก (Submit to Staging)
1. หลังจากผ่านหน้าความซ้ำซ้อนมาได้ ระบบจะให้คุณ **"กรอกอีเมลของคุณ"** (เพื่อใช้เป็น Soft Identity ในการติดตามสถานะ)
2. กรอกอีเมลแล้วกด **"ส่งข้อมูลเข้าระบบ (Submit)"**
3. หน้าจอจะขึ้นข้อความแจ้งเตือนสีเขียวว่า **"ส่งข้อมูลสำเร็จ! รอผู้ดูแลระบบอนุมัติ"** พร้อมกับแสดง **Job ID** (เช่น `job-1234abcd`) 
   - *หมายเหตุ: ระบบจะจำ Job ID และอีเมลของคุณไว้ในเครื่องโดยอัตโนมัติ*

---

## 🛡 เฟส 4: กระบวนการตรวจสอบโดยแอดมิน (Admin Approval)
ขั้นตอนนี้จำลองว่าคุณเป็น Admin ที่มีสิทธิ์อนุมัติข้อมูลเข้าสู่ระบบฐานข้อมูลจริง
1. คลิกที่เมนู **"Admin Dashboard"** (หรือไอคอนรูปโล่) ที่มุมบนขวาของหน้าจอ
2. ระบบจะถามหา **รหัสผ่านผู้ดูแลระบบ (Admin Key)**
   - ให้กรอกรหัสผ่าน (ค่าเริ่มต้นคือ: `fondue-admin-secret-999`)
3. คุณจะเห็นรายการ Job ทั้งหมดที่มีสถานะ `pending_approval`
4. **(ทางเลือก) ทดสอบตีตก (Reject):**
   - ลองกดปุ่ม **"Reject"** แล้วใส่เหตุผลว่า "ข้อมูลผิดพลาดตรง..."
   - *คุณสามารถกลับไปที่แท็บ "ประวัติของฉัน" เพื่อดูว่ามันขึ้นสถานะ Rejected พร้อมเหตุผลได้ (และสามารถกดปุ่มเพื่อแก้ไข Payload ใหม่ได้ด้วย)*
5. **ทดสอบอนุมัติ (Approve):**
   - คลิกที่ปุ่ม **"Preview"** เพื่อดูรายละเอียดหน่วยงานที่จะถูกสร้างหรือถูกผูก
   - หากแน่ใจแล้ว ให้กด **"Approve"**
   - งานจะถูกส่งให้ Background Worker ไปจัดการแทรกข้อมูลลงฐานข้อมูลจริง

---

## ✅ เฟส 5: ตรวจสอบสถานะการทำงาน (Execution Check)
1. กลับไปที่แท็บ **"ประวัติของฉัน (My Submissions)"**
2. ใส่อีเมลที่คุณใช้ใน เฟส 3
3. คุณจะเห็น Job ของคุณเปลี่ยนสถานะเป็น **"Processing"** (กำลังประมวลผล) หรือ **"Completed"** (เสร็จสมบูรณ์)
4. หากสถานะเป็น `completed` และ Progress Bar เต็ม 100% แสดงว่า **หน่วยงานทั้งหมดถูกสร้างลงในระบบจริงเรียบร้อยแล้ว!**

---

## 📸 เฟส 6: (ส่วนเสริม) การตรวจสอบผลลัพธ์ QR Code (Backend)
หากต้องการพิสูจน์ว่า Background Worker ทำงานสร้างภาพ QR Code ออกมาได้จริง (ไม่ต้องทำผ่าน UI):
1. เปิด Terminal ในฝั่ง `fondue-org-importer`
2. สามารถรันสคริปต์สุ่มเทสต์ได้เลย:
   ```bash
   go run scratch_sample.go
   ```
   หรือ 
   ```bash
   go run scratch_sample_report.go
   ```
3. สคริปต์จะจำลองการสร้าง QR Code และบันทึกไฟล์ภาพ .png ลงในโฟลเดอร์ของโปรเจกต์ ซึ่งคุณสามารถเปิดดูรูป QR Code ที่ผ่านการวาง Layout สวยงามได้ทันที!
